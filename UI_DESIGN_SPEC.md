# Channel UI Design Spec

本规范基于当前 `stitch_desktop_community_channel_ui/_2/code.html` 的桌面版页面整理，用于后续频道相关页面的统一设计。

目标不是“复刻一个页面”，而是抽出这套界面的稳定规则，保证未来新页面看起来属于同一产品。

## 1. 设计目标

- 桌面端以“内容浏览 + 轻操作 + 低打扰”为核心。
- 信息层级清楚，用户一眼能分辨：
  - 全局导航
  - 当前频道
  - 内容输入入口
  - 板块/排序切换
  - 内容流
  - 辅助信息栏
- 优先做“安静的界面”，避免无意义的视觉炫技。
- 复杂性由系统吸收，用户尽量少做判断。

## 2. 页面骨架

当前页面采用稳定的三段式桌面结构：

- 左侧：全局侧边导航
- 中间：频道主内容区
- 右侧：辅助信息区

推荐固定结构：

- 左侧导航宽度：`256px`
- 中间内容最大宽度：`768px` 左右
- 右侧辅助栏宽度：`288px`
- 主内容容器最大宽度：`1200px`
- 页面左右留白：桌面端 `24px - 32px`

布局原则：

- 中间内容永远是视觉主轴
- 右侧栏不能抢主内容注意力
- 左侧导航是系统层，不参与当前频道内容层级竞争

## 3. 视觉基调

整体气质：

- 深色、克制、安静
- 内容优先，不做高饱和大面积干扰
- 用少量蓝色承担“当前状态 / 主行动 / 链接”

基础颜色：

- 页面背景：`#1A1A1A`
- 深色表面：`#242424`
- 边框：`#333333`
- 主文字：`#E5E5E5`
- 次文字：`#A3A3A3`
- 主强调色：`#3B82F6`

使用规则：

- 主色只用于：
  - 当前 tab
  - 主要按钮
  - 勾选
  - 可点击强调链接
- 大面积背景尽量不用纯主色
- 辅助卡片和弹窗优先使用深灰层级，而不是重新定义新颜色

## 4. 圆角、边框、阴影

当前页面的圆角体系是统一的，后续页面不要随意跳级。

推荐等级：

- 小控件圆角：`8px - 12px`
- 常规卡片圆角：`16px`
- 大容器/频道头图圆角：`20px - 24px`
- 胶囊按钮：`9999px`

边框规则：

- 统一使用细边框
- 默认边框色：`#333333`
- 不要在同一层叠加“粗边框 + 发光 + 重阴影”

阴影规则：

- 常规卡片尽量无重阴影
- 只有浮层/弹窗可以使用适度阴影
- 阴影是为了区分层级，不是为了装饰

## 5. 字体与信息层级

不要把文字做得过大。当前 UI 的正确感觉来自“偏紧凑而清楚”的字级关系。

推荐字级：

- 页面主标题 / 频道名：`24px`
- 次级标题 / tab：`16px - 18px`
- 正文主内容：`15px`
- 次级信息 / 作者时间 / 辅助说明：`12px - 14px`
- 浮层菜单项：`15px`

字重规则：

- 标题：`600 - 700`
- tab / 主要操作：`500 - 600`
- 正文：`400 - 500`
- 元信息：`400`

使用原则：

- 同一块区域内不要出现过多字号
- 优先用字重和颜色区分层级，不要靠无限放大

## 6. 间距系统

当前页面的舒适感主要来自一致的 8pt 间距节奏。

推荐间距：

- 超小：`4px`
- 小：`8px`
- 中：`12px`
- 常规：`16px`
- 大：`24px`
- 模块间距：`32px`

应用方式：

- 文本与图标常用 `8px`
- 小组件内部常用 `12px - 16px`
- 卡片内边距常用 `16px - 24px`
- 大区块之间用 `24px - 32px`

禁止：

- 同一层级出现随机留白
- 某个弹窗或卡片因为内容少就放大外边距撑体积

## 7. 组件规范

### 7.1 左侧导航

组成：

- 产品标识
- 搜索框
- 导航分组标题
- 分组条目
- 当前频道高亮
- 当前用户信息

规则：

- 导航条目高度保持一致
- 当前项用轻背景高亮，不用高饱和色块
- 图标尺寸统一
- 分组标题明显弱于导航项

### 7.2 频道头图

组成：

- 频道 logo
- 频道名
- 成员数
- 右上操作 icon

视觉规则：

- 头图是页面最强视觉块，但只占页面顶部一块
- 使用柔和渐变或轻装饰，不要喧宾夺主
- logo、标题、操作区要形成稳定左右结构

收缩规则：

- 下滑后头图压缩成紧凑头栏
- 保留：logo、频道名、右上 icon
- 去掉：大面积 banner 视觉、成员数等次信息

### 7.3 发帖输入区

有两种状态：

- 展开态：完整发帖框
- 折叠态：轻量入口

规则：

- 页面默认进入折叠态
- 下滑后不再保留大输入框
- 折叠后由板块切换 + 小号 `发帖` 按钮承担操作入口

### 7.4 板块切换区

组成：

- 左侧排序/筛选 icon
- tab 项
- 收缩态下右侧 `发帖`

规则：

- tab 是轻量导航，不应做成厚重 segmented control
- 当前项只用下划线 + 主色表示
- 下划线只服务选中项，不要额外铺整条底线

### 7.5 公告栏

定位：

- 属于右侧辅助信息栏
- 永远弱于中间内容流

规则：

- 卡片尺寸固定、视觉安静
- 顶部与左侧中栏的“输入区/板块区”保持同层起点
- 滚动收缩时，公告栏跟随顶部结构一起上移
- 公告栏与顶部区之间的间距要稳定，不要在滚动来回后漂移

### 7.6 内容流

帖子卡片规则：

- 作者信息一行
- 正文一块
- 操作区一行
- 卡片间通过分割线区分，不依赖大背景块

原则：

- 内容流是页面主体
- 操作按钮保持轻量
- 卡片不需要额外厚重背景，边界靠留白和分割线建立

### 7.7 浮层与弹窗

适用对象：

- 消息弹窗
- 频道菜单
- 排序筛选弹窗

统一规则：

- 尺寸必须参考周围控件和字级，不单独长成“大面板”
- 弹窗优先小而准，不要一上来做成半屏
- 保持外边距，不贴页面底部或边缘
- 弹窗只承担当前任务，不叠加无关信息

筛选弹窗特殊规则：

- 挂在触发 icon 下方
- 宽度接近 `160px`
- 文案字号约 `15px`
- 每项高度偏紧凑
- 只显示少量选项，不做复杂菜单

## 8. 滚动与吸顶规范

这是当前页面最重要的交互规则之一。

正确模型：

- 顶部区域下滑后会压缩
- 不是弹出一个全新顶栏
- 是原组件自身收缩成更轻量的操作层

滚动后固定在顶部的应该只有：

- 紧凑头栏
- 板块切换
- 小号 `发帖`
- 右侧公告栏

不应该固定的：

- 原大头图背景壳
- 大输入框
- 多余的毛玻璃背景
- 跨整行的无意义底线

原则：

- sticky 的对象应是“收缩后的最小操作层”
- 不要固定原顶部大容器
- 不要让背景层覆盖到右侧辅助栏

## 9. 比例控制原则

这是未来复用时最需要遵守的规则。

新页面新增 UI 时，先看周围已有元素，再决定尺寸。

必须优先参考：

- 当前页面正文大小
- tab 大小
- 右上 icon 大小
- 输入框高度
- 右侧卡片尺度

判断标准：

- 如果一个新浮层一眼看上去比周围控件大一圈，通常就是错的
- 如果一个交互组件需要靠“很大”才能显眼，通常说明设计方式错了

## 10. 新页面复用建议

未来做其他页面，优先复用这套模板：

### 频道内容页

- 左侧导航
- 中间频道头图 / 内容流
- 右侧公告或辅助卡片

### 频道详情页

- 保留顶部频道头栏
- 中间改为介绍、成员、规则、资源
- 右侧保留辅助信息卡片

### 频道管理页

- 保留同样的顶部频道识别区
- 中间替换为设置表单/管理列表
- 右侧改为帮助说明、状态提示

### 频道消息页

- 保留频道识别和操作 icon
- 主内容区切为消息列表
- 仍沿用当前浮层和字级规则

## 11. 禁止事项

未来页面设计中，不要做这些事：

- 不要新增明显偏大的弹窗或浮层
- 不要让浮层尺寸脱离周围控件比例
- 不要让 sticky 区保留大块残余背景
- 不要让右侧辅助栏被中间层背景覆盖
- 不要随意增加新的圆角、阴影、颜色体系
- 不要用大面积装饰抢走内容注意力
- 不要在一个页面里混入两套不同密度的控件

## 12. 一句话设计准则

这套 UI 的本质不是“炫”，而是：

`深色、安静、比例克制、内容优先、交互收缩后仍然清楚。`

以后做新页面时，只要新页面仍然满足这句话，基本就不会跑偏。
